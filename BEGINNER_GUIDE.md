# 👶 初心者向け - コード修正ガイド

## このガイドについて
このシステムのコードを修正したい人へ、わかりやすく説明します。
英語が分からなくても、日本語のこの説明を読めば大丈夫です！

---

## 📝 コードを修正した場合の流れ

### 【ステップ1】GitHub で修正したコードを保存する
```
1. GitHub にアクセス（URLを開いた状態）
2. ファイルを選んで、 [Edit] ボタンをクリック（鉛筆アイコン）
3. コードを修正する
4. 下まで スクロール して [Commit changes...] をクリック
5. 説明を書く（何を修正したか、日本語でOK）
   例：「ドラッグ&ドロップ機能を追加しました」
6. [Commit changes] ボタンをクリック
```

### 【ステップ2】X Server を開く
```
1. ファイルマネージャーを開く
   URL: https://webftp-sv10875.xserver.jp/
2. ログインする
3. public_html → org-chart フォルダを開く
```

### 【ステップ3】待つだけ（自動同期）⏳
```
⚠️ ここが大切！

GitHub にコードを保存すると、
自動的に X Server のファイルが更新されます！

何もしなくていい。
ただ待ってください。

時間：　数秒～数分で完了
```

---

## 🔄 自動同期の仕組み

```
GitHub で [Commit changes] をクリック
        ↓
GitHub が X Server へ自動的に通知
        ↓
X Server が自動的に最新のコードを取得
        ↓
soulsyncs.jp に反映される ✅
```

---

## ✅ 確認方法

修正が反映されたか確認したい場合：

```
1. soulsyncs.jp にアクセス
2. ブラウザを「更新」（キーボード：F5 キー）
3. 修正が反映されているか確認
```

---

## ⚠️ よくある失敗

### ❌ 「修正したのに反映されない」

**原因1：まだ反映されていない時間**
- 対策：数分待ってから、ブラウザを更新（F5キー）してください

- **原因2：ブラウザキャッシュ**
- - 対策：キーボードで「Ctrl + Shift + R」（Macなら「Cmd + Shift + R」）で強制更新
 
  - **原因3：GitHub でのコミットを忘れた**
  - - 対策：GitHub の「Commit changes」をもう一度クリック
